# v0.0.17 — functional validation (end-to-end)

The v0.0.17 engine — and the 0.0.16 improvements it bundles — verified **working
end-to-end** by exercising the built `praxec` binary directly (CLI), not just by
the test suite. Every check below was run against `target/debug/praxec` at the
`feat/v0.0.16-engine` tip.

## Governed connections (D4a) + the operator-grant boundary (D3)

- `praxec connections add slack --kind mcp …` → wrote the connection **staged /
  ungranted** under `stagedConnections:` (inert — never in the live registry).
- `praxec connections grant slack` → promoted it to the live `/connections`
  registry **and emitted a real `connections.granted` audit event**
  (`actor: operator`), and appended `slack` to `grant_connections:`.
- Fail-fast guards fired: a duplicate `add` → `DUPLICATE_CONNECTION`; a `grant`
  of an unstaged name → `CONNECTION_NOT_STAGED` (with the exact remedy). No path
  auto-grants — an agent/pack shelling out to `add` cannot silently obtain a
  trusted connection.
- `praxec check` validated the config with the granted connection live.

## Observability (0.0.16)

- `praxec schema audit-event` printed the published `AuditEvent` JSON Schema
  (code-first, derived from the canonical Rust struct).
- Driving a mission emitted the full lifecycle event stream to the file audit
  sink: `workflow.started`, `transition.requested`, `executor.started`,
  `executor.succeeded`, `workflow.transition`, `workflow.transitioned`,
  **`outcome.recorded`** (the event that feeds item1's intent-evidence), and
  **`workflow.completed`**.

## Governed runtime — mission driven to a terminal outcome

- A no-human-gate flow driven via the `praxec command` contract (the same
  contract the MCP `praxec.command` tool serves) over a sqlite store:
  `start` → `go` → `done`, ending at **`status = succeeded`** with outcome
  `finished` **`met = true`**.
- A human-gated flow (`examples/hello-flow.yaml`) correctly parked at
  `status = waiting` on its human gate — the gate holds for a proper mediator/HITL
  move rather than a plain submit (by design).

## Self-heal / poka-yoke (0.0.16)

- `praxec doctor` preflighted the machine: resolved required provider keys (none)
  and detected the staged connection's `npx` binary on PATH → `preflight: ok`.
- The config validator refused malformed flows — a state declaring
  `outcome: success` without `terminal: true`, and an unreachable terminal — with
  precise diagnostics, before any mission could start.

## Library surfaces (D1 / D2 / D4b / D6)

Covered by the 214-suite full-workspace gate (fmt + clippy `--all-targets` +
tests, 0 failures), including the tool-descriptor loader, tool-source executor,
`praxec.packs/v3` registry loader (mcp + cli + rest fixtures), and the
evidence+topology selector ranking with its explainable `why`.

## Still requires a running v0.0.17 MCP

Driving the **dogfood** deliverables (`flow.onboard-tool`, meta-proof) *through
the MCP tool surface* needs the operator to update the installed praxec MCP
binary (the CLI contract calls above already exercise the same runtime).
